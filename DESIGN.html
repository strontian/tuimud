tui-mud — Design Document

A terminal chat interface for Claude with a split-screen visual rendering area.

What is tui-mud?

tui-mud is a terminal-based chat interface for Claude (Anthropic's AI). It creates a split-screen TUI (Terminal User Interface) that combines a real-time chat area with a dedicated visual rendering area — similar in spirit to a MUD (Multi-User Dungeon), where an AI can describe, draw, and interact within a bounded terminal world.

  • Chat with Claude in a scrollable history pane
  • Claude can render visual content (ASCII art, diagrams, UI mockups) to a dedicated screen area
  • Claude can execute shell commands and read/write files on the local machine
  • Full keyboard navigation: type, scroll, resize-aware layout

Components

chat_tui.mjs
Entry Point

Minimal bootstrapper. Creates a ChatTUI instance and calls init().

lib/ui.mjs — ChatTUI
UI / Rendering

Owns the entire terminal display and all user input. Manages layout, ANSI rendering, keyboard events, and scroll state.

lib/client.mjs — MessageClient
API / Orchestration

Maintains conversation history, sends messages to Claude, and runs the tool-use loop until Claude returns a final text response.

tools/index.mjs
Tool Registry

Exports all tool schemas and executeTool(). Routes Claude's tool calls to the correct handler. Injects live screen dimensions into tool descriptions.

tools/screen.mjs
render_screen Tool

Accepts a content string from Claude, word-wraps it, and updates the visual pane. Each call replaces previous content entirely.

tools/shell.mjs
bash Tool

Runs an arbitrary shell command via execSync. Returns combined stdout/stderr (up to 10 MB).

tools/filesystem.mjs
File I/O Tools

Provides read_file, write_file, and edit_file (surgical string-replace with exact-match validation).

lib/ui.mjs — ChatTUI

  • Layout calculation: Divides terminal rows into header, screen area, chat area, and input row. Recalculates on resize.
  • Rendering: Writes ANSI escape sequences directly to process.stdout to draw borders, color-coded text, and cursor positioning.
  • Input handling: Enables raw mode on process.stdin. Handles printable characters, Backspace, Enter, arrow keys (scroll), and Ctrl+C.
  • State: inputBuffer, chatHistory[], screenContent[], scrollOffset, screenScrollOffset, isWaiting

lib/client.mjs — MessageClient

  • Message history: Maintains messages[] in Anthropic API format across the full conversation.
  • API calls: Sends requests to claude-sonnet-4-20250514 with max_tokens: 8096.
  • Tool-use loop: After each response, checks stop_reason. If tool_use, extracts tool calls, dispatches via tools/index.mjs, collects results, and sends back tool_result blocks. Repeats until done.
  • UI bridge: Holds a reference to ChatTUI to call addMessage() and read current screen dimensions.

Data Flow: User Message → Reply

  1. User types text and presses Enter
  2. ChatTUI.sendMessage() adds the message to chatHistory, calls MessageClient.sendMessage(text)
  3. MessageClient appends {role: 'user', content: text} to messages[] and calls the Anthropic API
  4. API returns a response. If stop_reason === 'tool_use':

    Extract each tool_use block (name + input)

    Call executeTool(name, input, ui) → dispatches to correct handler

    render_screen updates the visual pane immediately; bash / filesystem tools return result strings

    Append tool_use + tool_result messages to history, call API again

    Repeat until stop_reason !== 'tool_use'

  5. Extract final text response, call ChatTUI.addMessage('claude', text)
  6. ChatTUI.render() redraws the full terminal
